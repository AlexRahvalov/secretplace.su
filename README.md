# SecretPlace.su - Сайт для Minecraft сервера

Проект сайта для Minecraft сервера с динамически обновляющейся информацией и собственной реализацией компонентов.

## Структура проекта

```
secretplace.su/
├── client/                   # Фронтенд
│   ├── src/
│   │   ├── components/       # Переиспользуемые компоненты
│   │   │   └── Auth/         # Компоненты аутентификации
│   │   ├── pages/            # Страницы
│   │   ├── services/         # API сервисы
│   │   ├── utils/            # Утилиты
│   │   ├── assets/           # Статические ресурсы
│   │   ├── styles/           # Стили
│   │   ├── config.js         # Конфигурация клиента
│   │   └── main.js           # Точка входа
│   ├── public/               # Публичные файлы
│   └── package.json          # Зависимости клиента
├── server/                   # Бэкенд
│   ├── src/
│   │   ├── controllers/      # Контроллеры (MVC)
│   │   ├── models/           # Модели данных
│   │   ├── routes/           # Маршруты
│   │   ├── utils/            # Утилиты
│   │   ├── auth/             # Система аутентификации
│   │   │   └── adapters/     # Адаптеры аутентификации
│   │   ├── config/           # Конфигурационные файлы
│   │   ├── db/               # Работа с базой данных
│   │   ├── telegram/         # Интеграция с Telegram
│   │   ├── server.js         # Express-сервер
│   │   └── index.js          # Точка входа
│   ├── sql/                  # SQL-скрипты
│   └── package.json          # Зависимости сервера
├── package.json              # Основные зависимости
└── README.md                 # Документация
```

## Технологии

- **Фронтенд**: Vanilla JavaScript, Vite для сборки
- **Бэкенд**: Node.js, Express.js
- **База данных**: MariaDB (основная), SQLite (для локальной разработки)
- **Аутентификация**: Собственные адаптеры для EasyAuth, AuthMe, Telegram

## Запуск проекта

### Установка всех зависимостей

```bash
npm run install:all
```

### Запуск в режиме разработки

```bash
npm run dev
```

### Запуск только клиентской части

```bash
npm run client
```

### Запуск только серверной части

```bash
npm run server
```

## Архитектура проекта

### Клиентская часть

- **Компонентная архитектура**: Компоненты, реализованные на чистом JavaScript
- **Роутинг**: Собственная реализация роутинга на клиенте
- **Сервисы**: Отдельные сервисы для работы с API

### Серверная часть

- **MVC**: Структура Model-View-Controller
- **API**: RESTful API для взаимодействия с клиентом
- **Адаптеры**: Система адаптеров для различных способов аутентификации
- **База данных**: Поддержка различных СУБД через абстракцию

## Особенности

- Динамическое обновление информации о сервере
- Интеграция с различными авторизационными системами
- Адаптивный дизайн
- Интеграция с Telegram для авторизации и уведомлений
